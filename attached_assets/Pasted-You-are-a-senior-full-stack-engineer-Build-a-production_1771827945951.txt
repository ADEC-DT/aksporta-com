You are a senior full-stack engineer.
Build a production-quality MVP web application called:
STABLEMASTER
Equestrian Stable Management System
IMPORTANT:
This is an operational stable management system.
NetSuite remains the financial ERP.
Do NOT replicate ERP complexity.
The application must stay simple, fast, and operational.
====================================================
GOAL OF THE MVP
====================================================
The system manages:
- Stables (big buildings)
- Boxes (horse stalls, storage, tack rooms)
- Horses
- Customers
- Livery Agreements (recurring boarding contracts)
- Ad-hoc billable services
- Monthly billing preparation
- Invoice draft generation
The system DOES NOT replace accounting.
It prepares billing data later sent to NetSuite.

====================================================
DATA MODELS
====================================================
-------------------
ITEMS (READ ONLY)
-------------------
Items are mastered in NetSuite.
Users cannot create or edit them.
ItemService:
- id
- netsuite_item_id
- description
- default_unit_price
- base_unit
- category
- is_active
Items page behavior:
- Add button greyed out
- Message:
  "Items are mastered in NetSuite. Read-only list."
- Columns:
  Description | Category | Base Unit | Default Price | Active
- Filters:
  description search
  category dropdown
  active status
NO editing allowed.
-------------------
STABLES
-------------------
A Stable = big building containing many boxes.
Stable:
- id
- name
- notes
- is_active
CRUD allowed.
-------------------
BOXES
-------------------
Each box belongs to ONE stable.
Box:
- id
- stable_id (required)
- name (A01, B12…)
- box_type:
    STALL
    STORAGE
    TACKING_ROOM
- status:
    AVAILABLE
    OCCUPIED
    MAINTENANCE
- notes
Boxes page:
List all boxes across stables.
Filters:
- Stable
- Box Type
- Status
- Search by name
Users must choose a Stable when creating a Box.
Relationship:
Stable 1 → many Boxes
-------------------
HORSE
-------------------
Horse:

- remarks
- status
CRUD allowed.

CUSTOMER

CRUD allowed.

LIVERY PACKAGES

CRUD allowed.
-------------------
LIVERY AGREEMENTS
-------------------
Represents box rental.
IMPORTANT FEATURE:
Support agreements WITHOUT horse.
This is called:
UNASSIGNED AGREEMENT
(Customer rents box but rotates horses.)
LiveryAgreement:
- id
- stable_id
- box_id
- agreement_type:
    PERMANENT_AUTO_RENEW
    TEMPORARY
- start_date
- end_date (only temporary)
- customer_id
- horse_id (nullable)
- reference_number
- livery_package_id
- notes
Rules:
- horse_id optional
- display "(Unassigned)" when empty
- creating agreement reserves the box
  → box status becomes OCCUPIED even if no horse.
New Agreement page:
Show box occupancy table.
Columns:
Stable | Box | Type | Status | Horse | Customer | Action
Filters:
- Stable
- Available only toggle
Create button appears ONLY on available boxes.
Modal preselects stable + box.
-------------------
POST BILLING ELEMENT
-------------------
Used for ad-hoc services.
Replace old design.
Page shows searchable BOX list.
Columns:
Stable | Box | Type | Status | Horse | Customer | Action
Filters:
- Stable
- Horse search
- Customer search
- Status
NO “last 10 records”.
Click “Create Billing Element”:
Modal form:
- Stable (locked)
- Box (locked)
- Transaction Date

- Horse (optional)
- Item (NetSuite list searchable)

- customer_id
- item_id
- quantity
- unit_price
- transaction_date
- remarks
- billed=false

AUTOMATIC MONTHLY LIVERY LINES
-------------------
For each active agreement:
Generate virtual billing line per month.
Rules:
If agreement active during month:
→ create billing line:
quantity = 1
price = package monthly_price
date = first day of month
These lines appear as UNBILLED.
-------------------
INVOICE GENERATION
-------------------
User selects billing lines.
Constraints:
- Must belong to ONE customer.
Button:
Generate Invoice
Flow:
1. Validate same customer
2. Confirmation modal
3. Create Invoice Draft
Invoice:
- id
- customer_id
- date
- total
- status="DRAFT"
- lines[]
Mark selected lines as billed.
Show invoice list below table.
NO NetSuite integration yet.
