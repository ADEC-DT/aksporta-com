DATA MODEL (MVP)
Use these TypeScript models:

Facility:
- id, name, type: "STABLE" | "ARENA" | "OTHER"
- parentFacilityId?: string (to support parent/child)
StableBlock:
- id, facilityId, name (e.g., “Block A”, “Barn 1”)
StableUnit (stall/storage unit):
- id, stableBlockId, name/number (e.g., “A01”), unitType: "STALL" | "STORAGE"
- status: "AVAILABLE" | "OCCUPIED" | "MAINTENANCE"
- currentHorseId?: string (if OCCUPIED)

Horse:
- id, name, dob?, remarks?, color?, sex?, group?, status: "ACTIVE" | "INACTIVE"
Customer:
- id, name, email?, phone?, remarks?, status: "ACTIVE" | "INACTIVE"
ItemService:
- id, name, category: "SERVICE" | "ITEM"
- unitOptions: string[] (e.g., ["each","day","month"])
- defaultUnit: string
- unitPrice: number
- isActive: boolean
BillingElement (customer billing element):
- id
- stableBlockId, stableUnitId
- transactionDate (ISO date)
- refNumber (string)
- remarks (string)
- horseId
- customerId
- itemServiceId
- unit (string)
- unitPrice (number, auto from item)
- quantity (number, decimal)
- computedLineTotal (unitPrice * quantity)

LiveryPackage:
- id, name, monthlyPrice
- coveredItemServiceIds: string[]
LiveryAgreement:
- id
- agreementType: "PERMANENT_AUTO_RENEW" | "TEMPORARY"
- startDate, endDate? (end required for TEMPORARY)
- facilityId, stableBlockId, stableUnitId
- horseId
- customerId
- customerContact (string)
- refNumber (string)
- liveryPackageId
- notes/remarks
